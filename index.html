Get Cooking

Eating a well-rounded diet is key to leading a long, healthy life and the internet is chock-full of delicious recipes to satisfy anyone's palette. That being said, I don't need to read about the history of salmon before I learn how to prepare a fillet and I don't want to sign up for a mailing list. Traditional recipe sharing websites are cluttered with content that users simply don't care about and it ruins the experience.

Get Cooking is seeking to change the way people find and share recipes online by offering a clean, minimalistic user-interface with no blogs or excessive advertising. You are able to store your own recipes and keep a list of your favorites from other users. If you're looking for something specific, you can easily filter through a variety of tags or search by ingredient. If you aren't sure what to make, you can flip through the randomizer and see what catches your eye.

Check out it out at getcooking.app

Vulcan Stake Pool

What is Cardano?

Cardano is a blockchain platform for "changemakers, innovators, and visionaries" designed to bring positive change to the world. It operates on a proof-of-stake protocol, Ouroboros, that offers improved security guarantees and lower energy costs when compared to proof-of-work protocols.

What is a stake pool?

Owners of ADA (Cardano's native currency) have a stake in the network proportional to their holding size. ADA holders can earn rewards by delegating their stake to a stake pool. The more ADA delegated to a given stake pool, the higher the pool's chance to mint the next block for the blockchain. The stake pool operators and their delegators share the rewards earned by adding blocks.

What is Vulcan's mission?

Our mission is to empower Cardano as an emerging technology by operating as a node in the network, engaging with the vibrant community, and supporting decentralized finance (DeFi) applications built on the protocol. Currently, 25 ADA per epoch that Vulcan successfully mints a block will be pooled to create "The Vulcan Fund". This fund will strictly be used to support DeFi on the Cardano network through peer-to-peer lending.

Learn more at vulcanstakepool.com

Course Projects

Course: Capstone Project

Objective: Our objective is to create a fun, educational application for investing in the stock market. Stock Overflow will have two main features: an educational mode where users can practice investing in stocks with real-time prices and a competitive fantasy stock trading league where users can compete against their friends to see who can make the highest gain over a set period of time. All of the money and purchases are fictional of course, but will be simulating real-life investing as close as possible given our API limitations.

Technologies: This is a four person project that we'll be developing over the course of thirteen scrum sprints. We will manage our sprints with Confluence and JIRA, and utilize a Git repository hosted in BitBucket. The application will be written in Java using both the Spring and Hibernate frameworks. The frontend will be built with JavaServlet Pages (JSP), and data will be stored in a MySQL database.

Course: Advanced Web Application Development

Objective/Outcome: I sought to design a social media based gift giving platform. The idea is that users can add items to a list of gifts they're interested in, then their friends could "claim" gifts to buy for them for the holidays. The application would resemble a wedding registry, but wouldn't be limited to the wedding space. I completed the project on my own and, although the homepage is rough, I met all my desired requirements and am happy with the result.

Technologies: I completed this project individually using Subversion and an SVN repository for version control. The frontend html pages were built with JavaServlet Pages (JSP) and CSS styling was done with the Materialize framework. The server side code is written in Java with the spring framework and data is stored is a MySQL relational database hosted on Google App Engine (of course, the dev database was local to my machine). I used Maven for build automation and I successfully deployed the application to Google App Engine.
